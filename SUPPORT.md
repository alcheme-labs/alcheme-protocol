# Support

## Where to Ask for Help

- Use GitHub Issues for reproducible bugs and concrete implementation problems.
- Use GitHub Discussions for questions and design conversations if Discussions is enabled.
- If Discussions is not enabled, open an issue and clearly label it as a question or support request.

## What Helps Maintainers Triage Quickly

- repository path or package name involved
- exact commands run
- logs or screenshots for failures
- whether the problem reproduces on a clean checkout
- what you already tried

## Scope

This project is still being opened in phases. Public support is best-effort and focused on:

- build and test failures
- protocol and SDK correctness
- clearly reproducible frontend and service bugs

General product strategy, private deployment guidance, and internal roadmap questions may not be answered in public channels.

Because the first public snapshot intentionally excludes some internal docs and runbooks, it helps maintainers a lot if you restate any missing assumptions or expected behavior directly in the issue.
